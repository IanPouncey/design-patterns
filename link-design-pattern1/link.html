Inaccessible link design pattern

The link on this page has been created using a <span> element, with scripting to provide the expected behaviour. At first glance the link appears to function as expected, but this design pattern has serious accessibility flaws.

Note: When using HTML use the <a> element. Do not re-purpose other elements to create links.

Link example

W3C website

Link code


<span onclick="fetchResource(event);">W3C website</span>

Link notes

The <span> element is semantically neutral. It has no implicit role that informs assistive technologies what the element's purpose is. In other words there is nothing to inform assistive technologies that the link is a link. It is simply treated as plain text.

The <span> element is not a natively interactive element. This means the link cannot be focused on with the keyboard or targeted by speech recognition tools.

The link can be focused on with a screen reader using its virtual cursor (or equivalent), but cannot be activated using the keyboard.
