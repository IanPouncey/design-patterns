Inaccessible button example

The button on this page has been created using a <span> element, with scripting to provide the expected behaviour. At first glance the button appears to function as expected, but this design pattern has serious accessibility flaws.

Button code


<span id="button" class="default" onclick="javascript:toggleButton(event);">Foo</span>

Button notes

The <span> element is semantically neutral. It has no implicit role that informs assistive technologies what the control's purpose is. In other words, there is nothing to indicate to assistive technologies that the button is a button. It is simply treated as plain text.

The <span> element is not a natively interactive element, so the button can't be focused on with a keyboard or targeted with speech recognition tools.

The button can be focused on with a screen reader (using its virtual cursor (or equivalent)), but it cannot be activated with the keyboard. The enter key does nothing, and the space key fires its default action - scrolling the page.

The change in state of the button is purely cosmetic. There is no programmatic indication, meaning that screen reader users remain oblivious to the button's current state.
